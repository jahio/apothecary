# apothecary - proof of concept API for nationwide realtime pharmacy inventories

Why should you not know where whatever your doctor prescribed is - or isn't - in stock? Better yet, why shouldn't they be able to easily query with real time updates in your area? When this problem recently affected me, just to prove the point, I had AI generate an overly simplistic application to get a start on proving that with sufficient will, eschewing greed and political corruption, this is an absolutely solveable problem. I don't expect anyone will have the strength of character to actually enforce use of something like this (certainly not this specific codebase, definitely not in its present state - don't do that! This is AI-generated and proof-of-concept only!), but if nothing else, this proves that it can be done.

It furthermore proves that with a few tweaks, real-time price comparisons that are open to the public are absolutely possible.

This is only an API. It has no front-end consumer-facing component. Nothing pretty for people to look at. You'd need to be a programmer/engineer of some form to have any clue what's going on here (or have been exposed to that field in some way).

**WARNING: THIS CODE WAS AI-GENERATED AND LIKELY DOESN'T WORK.** I'll tweak it as I have time. I had it generated in high volume at the start just to prove the point that it was possible to start. The quality (or wisdom) of that decision is debatable - which proves another point about replacing human software engineers (particularly skilled ones) with AI. But I'll let the countless tales of "nothing works anymore!" sure to hit us all over the next decade be testament to that.
